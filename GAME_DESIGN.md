# GAME_DESIGN.md — The Game as a System

The mechanical design of the game: rules, numbers, and — most importantly — the
**content vocabularies** that define exactly how much freedom the Director has.
UX.md says how it feels, WORLD.md says what it means, TECH_SPEC.md says what it's
made of; this document says what it **is**.

Reading guide for numbers: values marked **[T]** are tunable starting points that
live in one config module and will move with playtesting and evals. Values marked
**[HARD]** are gauntlet-enforced bounds the Director can never exceed; changing one
is a protocol-version bump (TECH_SPEC §9). Shapes (formulas, vocabularies, state
structure) are design decisions — changing them requires editing this file first.

---

## 1. System Pillars

1. **Small legible integers.** HP in the tens, damage in single digits. Every
   number the player sees should be mentally computable from the log.
2. **Deterministic skeleton, seeded flesh.** Same seed + same manifests + same
   actions = same run, always. Variance exists but is seeded and narrow.
3. **Vocabulary over content.** The engine implements ~40 primitives; everything
   the player ever meets is a composition of them. New *content* never requires
   new *code*.
4. **Budgets, not vibes.** The Director spends explicit point budgets. Fairness is
   arithmetic before it is judgment.
5. **Everything in the log.** A mechanic whose resolution can't be narrated in one
   log line is too complicated for this game.

## 2. Run Structure

| Parameter | Value | Notes |
|---|---|---|
| Run depth | 12 floors [T] | Floor 12 holds the Hoard (win condition) |
| Depth bands | Shallows 1–4 · Middle 5–9 · Lowest 10–12 | Maps to WORLD §8 arc and all band tables below |
| Win | Reach the Hoard on floor 12, take one thing, run ends in `WIN` | |
| Loss | HP reaches 0 → `LOSS` | Permadeath; diary + memory persist |
| Abort | Player quits run → `ABORTED` | Recorded, remembered |
| Per-floor soft cap | 800 turns [T] | Then the Deep grows bored: escalating reinforcement spawns every 100 turns |
| Per-run hard cap | 8,000 turns [HARD] | Engine forces terminal state; finiteness invariant (NORTH_STAR §4.1) is mechanical, not hoped for |

Floor geometry is rooms-and-corridors (classic Mystery Dungeon), generated by the
engine from manifest *parameters* — the Director chooses knobs, never tile-by-tile
layouts:

| Band | Grid size [T] | Rooms [T] | Layout flavors available |
|---|---|---|---|
| Shallows | 32×20 | 4–7 | open, warren |
| Middle | 40×24 | 5–9 | open, warren, halls, ring |
| Lowest | 36×22 | 3–6 | halls, ring, sanctum (single composed set piece) |

Every floor is guaranteed by generation + Gate 2: stairs reachable from entrance,
all quest objectives reachable, no entity placed in unreachable cells [HARD].

## 3. Time, Turns, and Action Economy

- Strict turn cycle: player acts → all other actors act in deterministic order
  (stable actor-id order) → end-of-turn ticks (statuses, hunger, regen).
- One action per actor per turn: move 1 tile (8-way), attack, use item, pick up,
  talk, wait, descend. No action points, no speed stats in MVP — `haste`/`slow`
  statuses grant/deny every other turn, which is the only speed mechanic.
- Tick order within end-of-turn is fixed and documented in the engine: damage-over-
  time → durations decrement → hunger → natural regen. Deterministic and testable.

## 4. The Player Character

State: **HP**, **Level**, **XP**, **Fullness**, inventory, equipment, statuses.
Attack and Defense are derived: base (from level) + weapon/armor + buffs.

| Stat | Start [T] | Growth [T] | Cap [HARD] |
|---|---|---|---|
| HP | 20 | +4 per level | 99 |
| Level | 1 | XP from kills; XP to next level = 8 × current level [T] (≈3–4 band-typical kills per level) | 12 |
| Base ATK | 2 | +1 per 2 levels | — |
| Base DEF | 0 | +1 per 3 levels | — |
| Fullness | 100 | −1 per 10 turns [T]; at 0, lose 1 HP per 2 turns | 100 (200 briefly when overfed) |

- **Natural regen:** +1 HP per 6 turns [T] while Fullness > 0 — the Shiren rhythm:
  walking is healing, hunger makes walking expensive.
- Level and everything else resets between runs. Cross-run persistence is memory
  and diary only (WORLD §7) — no meta-progression of power. **[HARD]**
- Inventory: 16 slots [T]; identical consumables stack to 5. Equipment: 1 weapon,
  1 armor, 2 charms.

## 5. Combat Math

One formula, no exceptions:

> **damage = max(1, (ATK − DEF)) , then ×(0.85–1.15) seeded variance, rounded**

- Hit chance: flat 95% [T] for everyone (a miss is rare punctuation, not a build).
- No critical hits, no damage types, no elemental table in MVP. Damage is damage.
  (Statuses carry all the tactical flavor — a `burn` proc *is* the fire sword.)
- Ranged attacks (bolt shape) use the same formula; line of sight required.
- Every resolution logs: attacker, defender, roll context, result, in one line.

## 6. Status Vocabulary [HARD — the closed list]

The complete set of statuses in the game. The Director composes freely *from* this
list and may never invent outside it. Durations in turns; all bounds [HARD].

| Status | Effect | Duration bound |
|---|---|---|
| `poison` | −1 HP per turn; stops at 1 HP (poison never kills) | 3–10 |
| `burn` | −2 HP per turn; can kill; cured by `cure_status` or water tiles | 2–5 |
| `regen` | +2 HP per turn | 3–10 |
| `stun` | Skip turns | 1–2 |
| `confusion` | Moves/attacks go in random direction | 2–6 |
| `slow` | Act every other turn | 3–8 |
| `haste` | Extra action every other turn | 3–8 |
| `blind` | Vision radius 1 | 3–8 |
| `shield` | +3 DEF | 3–10 |
| `weaken` | −2 ATK (min 1) | 3–8 |

Stacking: re-application refreshes duration, never stacks magnitude. `haste`+`slow`
cancel. Max 4 concurrent statuses per actor; oldest falls off.

## 7. Effect Vocabulary [HARD — the Director's verbs]

The atomic effects from which **all** items, enemy abilities, traps, and shrine-like
interactions are composed. ~16 verbs; each takes bounded parameters. This list is
the single most load-bearing table in the project: the engine implements exactly
these, and the Director's creativity is names + fiction + composition + parameters.

| Verb | Parameters (bounds [HARD]) | Notes |
|---|---|---|
| `damage` | amount 1–12 | |
| `heal` | amount 1–20 | |
| `apply_status` | status (from §6), duration (per §6) | |
| `cure_status` | one status or `all` | |
| `buff_stat` | ATK or DEF, ±1–3, duration 5–20 | |
| `nutrition` | +10–100 Fullness | |
| `teleport_self` | random walkable cell | escape valve |
| `teleport_target` | random walkable cell | |
| `blink` | directed hop 2–4 tiles | |
| `knockback` | push 1–3 tiles, collision = 1–3 dmg | |
| `reveal` | map / items / enemies / traps (this floor) | |
| `identify` | one carried item, or all of one category | |
| `enchant` | weapon or armor, +1 (item cap +3) | |
| `summon` | 1–3 entities **from this floor's roster**, adjacent cells | no novel entities mid-floor |
| `transform` | re-roll target enemy into roster entity of ≤ equal budget cost | |
| `dig` | remove wall tiles in a line, length 1–5 | |

**Composition rules:** an item or ability bundles **1–3 effects** + one **trigger**
+ one **targeting shape**.

- Triggers: `quaff`, `read`, `throw_hit`, `equip_passive`, `on_hit` (weapon proc,
  chance 10–30% [HARD]), `on_struck` (armor proc, same), `step` (trap), `use`
  (tool, 1–5 charges).
- Targeting shapes: `self`, `melee`, `bolt` (first target in line, range 3–8),
  `burst` (radius 1–2, centered self or impact), `floor`.

Growing this vocabulary is the **intended** post-MVP expansion axis: add a verb to
the engine (with tests), bump the protocol version, and the Director's expressive
space multiplies. Verbs are added by humans via this file, never by the Director.

## 8. Items and the Economy

**Categories:** weapon, armor, charm (passive trinket), draught (quaff), note
(read), throwable, food, tool (charges), key/quest item, coin.

| Category knob | Bound |
|---|---|
| Weapon ATK bonus | +1–6 [HARD], enchantable to +3 over |
| Armor DEF bonus | +1–5 [HARD], same |
| Charm | exactly one `equip_passive` effect |
| Item value | Shallows 5–30 · Middle 20–80 · Lowest 60–200 coin [T] |
| Items per floor | 4–8 [T], of which ≥1 food on every floor in Shallows/Middle [HARD — anti-starvation floor] |

- **Identification:** draughts, notes, and charms spawn unidentified, with a
  per-run seeded appearance pool ("a chalky draught", "a humming charm").
  Identify-by-use; knowledge lasts the run. Weapons/armor reveal their bonus only
  when equipped; cursed (negative) gear exists at low rate (≤10% [T]) and announces
  itself on equip — fair (WORLD §3.2), still costly.
- **Economy:** coin exists for barter with the Kept. Merchants buy at 50% value,
  sell at 100–150% [T]. Quest rewards: 1.5–3× the floor's item-value band, paid in
  coin, items, or identification — never in stat ups or rule-breaks [HARD].
- The Director prices Made items within the band tables; Gate 1 rejects outliers.

## 9. Enemy Design: Budgets and Behaviors

An enemy = **stat block** (within band budget) + **1–3 behaviors** (from §9.2)
+ **0–2 abilities** (composed per §7) + glyph + name + origin tag (Made / Old
Stock, per WORLD §4).

### 9.1 Stat budgets per band [HARD]

| Band | HP | ATK | DEF | XP yield | Max enemies alive per floor | Spawn budget per floor (points) |
|---|---|---|---|---|---|---|
| Shallows | 4–14 | 2–5 | 0–2 | 2–6 | 8 | 20 [T] |
| Middle | 12–30 | 5–10 | 1–4 | 5–14 | 12 | 45 [T] |
| Lowest | 24–50 | 9–16 | 3–7 | 12–25 | 10 | 70 [T] |

Each enemy's *cost* is a deterministic function of its stats, behaviors, and
abilities (pinned in the engine, tested). The Director fills a floor by spending
the spawn budget — fairness as arithmetic (Pillar 4). Abilities and behaviors add
cost; a cheap statline with a nasty ability is legal but not free.

### 9.2 Behavior vocabulary [HARD — the closed list]

| Behavior | One-line semantics |
|---|---|
| `approach_melee` | Pathfind to player, attack adjacent |
| `keep_range` | Hold distance band 2–5, use bolt ability when able |
| `flee_low_hp` | Below X% HP (20–50), run from player |
| `pack_hunter` | Waits until ≥N same-tag allies have line of sight (N: 2–4 [HARD]), then all engage |
| `ambusher` | Dormant/hidden until player within radius 1–2, then acts |
| `territorial` | Passive until player enters radius 2–4 or attacks |
| `guard` | Tethered to a cell/object within radius 1–3; never strays |
| `patrol` | Walks fixed waypoint loop until player sighted |
| `thief` | Steals one inventory item on hit, then flees; drops loot if killed |
| `caster` | Uses ability on cooldown 3–6 turns when condition met |
| `bodyguard` | Intercepts attacks aimed at a designated ally; protects NPCs/elites |
| `mimic` | Disguised as an item until interacted with |

Composition examples (what the Director actually does): *thief + flee_low_hp +
blink ability* = the infuriating pickpocket; *territorial + caster(apply_status:
burn, bolt)* = a turret you can route around; *pack_hunter + approach_melee* = wolf
floor. Twelve behaviors × sixteen verbs × parameters is a content space no team
could exhaust — Pillar 3.

## 10. Traps, NPCs, and Quests (mechanical view)

- **Traps:** hidden `step`-trigger effect bundles, revealed by `reveal traps`,
  adjacency, or triggering. 0–4 per floor [HARD]. Damage-class traps obey §7 bounds;
  a trap can never be lethal from full HP [HARD — computed at placement].
- **NPCs (the Kept):** 0–2 per floor [HARD]. Mechanically: dialogue trees of 2–5
  finite choices per node, ≤3 nodes deep; optional merchant inventory (≤6 items);
  optional quest. NPCs are invulnerable in MVP (no NPC-killing economy) — the Deep
  protects its collection.
- **Quest objectives [HARD — the closed list]:** `fetch` (item, this or next
  floor), `kill` (tagged target), `reach` (cell/feature), `deliver` (item →
  NPC), `escort` (NPC → stairs), `constraint` (floor-scoped dare: e.g., take no
  damage, kill nothing — checked by engine flags). One active quest per floor band,
  max 3 per run [T]; every quest completable in-run [HARD], rewards per §8.
  These six map onto WORLD §9's archetypes; the archetype is fiction, the
  objective is mechanics.

## 11. Difficulty Bands and the Fairness Gate

Gate 2 (NORTH_STAR §5) judges every floor with a bot ensemble before the player
sees it: 3 scripted policies (cautious / balanced / aggressive) × 5 seeds [T].
Starting thresholds, tuned by evals:

| Band | Clear rate (ensemble) | Median HP retention | Hard rejects |
|---|---|---|---|
| Shallows | ≥ 95% | 55–90% | any bot death on floors 1–2 |
| Middle | ≥ 85% | 30–75% | clear rate < 60% |
| Lowest | ≥ 70% | 15–60% | clear rate < 40% |

Also rejected regardless of band: floor solvable only with a specific consumable,
zero-threat floors below depth 2 (boredom is a defect too), and any floor whose
spawn budget, item values, or entity stats exceed the [HARD] tables — that's Gate 1.

**The difficulty *curve* belongs to the Director** (it may compose a breather floor
or a gauntlet floor deliberately); the *band* belongs to the engine. Freedom inside
fences, the project's whole thesis in miniature.

## 12. What the Director Actually Authors (the manifest, design view)

Per floor, the Director submits exactly: layout knobs (§2), theme/palette choice,
enemy roster + placements (within §9 budgets), item list + placements (§8), traps
(§10), 0–2 NPCs with dialogue/inventory/quest (§10), narration beats (floor intro
line + up to 3 [T] triggered observation lines), and metadata (origin tags,
callbacks referenced). Schema detail is a phase-plan concern; *this* table of
what-and-how-much is the design contract the schema must express.

Per run, additionally: one **signature moment** (NORTH_STAR §6) — a single
flagged floor in the Middle band where budgets relax by 25% [T] and a named Made
entity is permitted. One per run [HARD].

## 13. Deliberately Excluded Mechanics (MVP)

Decided against; relitigate by editing this file, not in a phase plan:

- No classes, races, builds, or skill trees — the run's items are the build.
- No damage types/elements, no crit system, no to-hit stats.
- No speed stat (statuses only), no diagonal-blocking rules, no facing.
- No crafting, no durability, no encumbrance beyond slot count.
- No meta-progression of power between runs (memory only).
- No boss-fight system in MVP — the signature moment carries the drama; a true
  finale fight for floor 12 is the first post-MVP candidate.
- No allies/pets/companions (escort quests are the only friendly-unit mechanics).

Each exclusion keeps the engine small enough that agents can hold all of it in
context — which, per TECH_SPEC's agent-operability constraint, is itself a feature.
