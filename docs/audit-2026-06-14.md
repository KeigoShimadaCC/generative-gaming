## 🔴 CRITICAL

### C1 — Director generation can permanently deadlock the whole pipeline
`src/director/orchestration/prefetch.ts:349` *(verified)*

`globalGenerationInFlight` is set (line 427) before `runGeneration` runs, but
`buildPrompt(depth, trace)` is called at line 349 **outside** the `try` whose `finally`
(388–393) releases the gate. `assemblePrompt` throws when the assembled prompt exceeds
`PROMPT_MAX_CHAR_LENGTH` (32 000). On that throw the gate is never released, so every
future `startPrefetch` short-circuits at the `globalGenerationInFlight !== null` guard
(417) — all subsequent floor generation is silently dead for the process lifetime,
degrading to fallback, and `cancel()` doesn't reset it either. Also surfaces as an
unhandled promise rejection. *Fix: move `buildPrompt` inside the `try`, or release the
gate in a `catch`.* Becomes plausible as cross-run memory accumulates into a large prompt.

## 🟠 HIGH

### H1 — Merchant barter refuses valid sells (common case)
`src/engine/npc/barter.ts:370` *(verified)*

`addCoinValue` refuses when `amount % unitValue !== 0`, but coin denominations are
`value.coin ≥ 5` while sell/credit prices are arbitrary `Math.round(...)` integers. A
price of 4 paid in a `coin:5` denomination → `4 % 5 ≠ 0` → spurious `insufficient_coin`.
Most ordinary transactions fail.

### H2 — Barter overpays and destroys coin value (no change given)
`src/engine/npc/barter.ts:343` *(verified)*

`removeCoinValue` pays a sub-denomination remainder by removing
`Math.ceil(remaining / value.coin)` whole coins. Affordability is checked on total value,
so the player passes the check then is silently overcharged (e.g. pay 4 with a 30-value
coin → 26 destroyed).

### H3 — SQLite foreign keys are never enforced
`src/harness/persistence/connection.ts:68` *(verified)*

`new Database(path)` issues no `PRAGMA foreign_keys = ON` (off by default in
better-sqlite3) and no `busy_timeout`/WAL. The `REFERENCES profiles(id)` on
`memory_events` is inert — orphan rows are accepted, and concurrent writers get immediate
`SQLITE_BUSY` with no retry anywhere.

### H4 — Director API routes accept unvalidated bodies & leak internals
`app/api/director/{start-generation,poll-status,get-floor}/route.ts:10–17` *(reported)*

Each route does `(await request.json()) as XxxRequest` with zero field validation — a
missing/non-numeric `depth` flows into `startPrefetch(depth+1, …)` as `NaN`, malformed
`trace` crashes deep in the assembler. The catch returns `error.message` verbatim with
**500** (e.g. `run <id> is not active`, filesystem paths). No auth, rate limit, or
body-size cap on the compute-heavy, state-mutating `start-generation` endpoint.

### H5 — Replay verification passes on truncated/incomplete traces
`src/harness/replay/replay.ts:31–67` + `parse.ts:71` *(reported)*

`replayTrace` returns `{status:"identical"}` if every present turn matches — with no
terminal-state/completeness check and no `engineVersion` comparison (only
`protocolVersion`). A trace truncated mid-write (the recorder has no fsync/atomic finalize
— `trace/recorder.ts:167`) is indistinguishable from a clean golden run. This is the
dangerous direction for a verification harness.

### H6 — Hard-cap LOSS overrides a winning final-turn action
`src/engine/turn/loop.ts:240–243` *(reported)*

`step()` checks `forcedTerminalFor` (forced LOSS at `turn ≥ 8000`) before
`applyPlayerAction`, so a winning `descend`/`abort` submitted on the cap turn is recorded
as a LOSS. The `take_hoard` win path bypasses this check, so the two win paths disagree.

### H7 — Quests silently wedge when the reward can't be paid
`src/engine/quests/machine.ts:300–315` + `objectives.ts:207` *(reported)*

If a coin reward can't fit (`addToInventory → illegal`), `completeQuest` returns
`illegal`, which every caller swallows (`if (!("illegal" in completed))`). The quest stays
`active` and silently re-attempts every turn with no player feedback. Coins quantize to
5/slot over 16 slots, so a near-full pack triggers it.

### H8 — `RunControllerRegistry` controllers are never evicted in production
`src/director/orchestration/transport.ts:60–100` *(reported)*

`getOrCreate(runId, …)` is called per start, but `remove` is only ever called from a test.
A long-lived server leaks a `PrefetchController` (with its `discards`, `completedByDepth`,
closures) per distinct `runId` — unbounded Map growth.

## 🟡 MEDIUM

- **Floor connectivity invariant is door-blind; enemy pathing is door-aware.**
  `floorgen/generate.ts:557–590,650` vs `map/path.ts:75`. `assertConnectivity` treats a
  closed door as walkable, so a door rolled onto a 1-wide chokepoint passes validation but
  partitions the map for enemy AI (`path()` defaults `openDoors:false`). Player can open
  doors → not a softlock, but degraded AI + replay perturbation.
- **`buff_stat` bypasses the status cap/refresh logic** (`effects/core.ts:751`,
  `appendStatusWithCap`): duplicate buffs stack additively up to the cap and silently
  evict index 0 with no event — ATK/DEF stacking exploit.
- **Derived `defense` returned unclamped** (`systems/combat.ts:186`): negative defense (via
  stacked `weaken`/negative buffs) makes incoming damage exceed the attacker's raw attack.
- **`index.json` read-modify-write race** (`artifacts/write.ts:199`): concurrent floor
  writes lose-update the index → a `generation.json` exists on disk but isn't indexed,
  later throwing "not indexed". The atomic temp+rename doesn't prevent the lost update.
- **Corrupt JSON crashes artifact readers** (`artifacts/read.ts:35,54`): unguarded
  `JSON.parse`; one bad `index.json` aborts enumeration of all runs (inside the `listRuns`
  loop). `content-loader.ts` wraps this; the artifacts layer doesn't.
- **`readJsonBody` casts unvalidated JSON to typed requests** (`transport.ts:107–155`):
  structurally-wrong bodies reach generation code, turning into generic 500s.
- **Banned-vocab regexes compiled without validation** (`gate3/heuristics.ts:184`): a
  malformed pattern in `content/banned-vocab.json` throws `SyntaxError` synchronously
  inside the Gate3 hook, aborting `generateFloor` for every floor (also recompiled per
  entry).
- **`simulate --seeds` allocates from an unbounded CLI int** (`cli/simulate.ts:357`):
  `--seeds 100000000` OOMs/hangs before validation; `>2³²` throws `RangeError`. No
  `Number.isSafeInteger`/upper bound.
- **Art-director artifact writes aren't isolated per sprite** (`artdirector/director.ts:181`):
  one sprite's write failure (disk full, `wx` collision) throws out of `generateSprites`,
  discarding the whole batch's accepted sprites; `generate-art.ts` then aborts.
- **`pollStatus` can report another depth's status across concurrent runs** (`poll-status`
  → `transport.ts:90`): mitigated by a client-side `response.depth === depth` guard →
  spurious fallback, not corruption.
- **Player level-up events log stale state** (`systems/player.ts:229`): multi-level-up uses
  the input `state` not `nextState`, so all level-up events carry pre-level-up HP/stats.
- **Charm equip overwrites slot 0 when all slots full** (`systems/inventory.ts:774`):
  `defaultCharmTarget` returns `index:0` instead of rejecting → silent charm loss.
- **`ArtifactViewer` forces a network round-trip mid-run** (`components/ArtifactViewer.tsx:42`):
  in-memory artifact model is never plumbed in (`artifactModel={null}`), so the Artifacts
  tab fetches `/api/artifacts` (likely empty for an in-progress run).

## 🟢 LOW (latents & polish)

- **Status nondeterminism latents:** several `[...new Set(...)].sort()` / `localeCompare`
  orderings (status, traps, quest-log, geometry targeting) are deterministic today but
  locale/engine-order-dependent; EntityIds sort lexically (`enemy#10 < enemy#2`) in a
  couple of spots — re-verify before relying on seeded replay.
- **Unbounded model-output gap:** `glyphSchema` (`schemas/common.ts:13`) has `.min(1)` but
  no `.max`/single-grapheme constraint — a 10k-char "glyph" flows into state/render while
  every other text field is capped.
- **32-bit FNV-1a as the replay verification hash** (`trace/hash.ts:9`): ~1-in-4.3e9
  per-turn collision reports a real divergence as `identical` — cheap to widen.
- **CLI `parseInt` accepts trailing garbage** (`simulate.ts:404`, `replay.ts:197`):
  `--max-turns 50abc` → 50 silently; the eval runner's parser guards this, these don't.
- **Audio SFX gain node never `disconnect()`ed** (`audio/useGameAudio.ts:195`) — bounded
  per-batch leak until context close.
- **WebGL context loss unhandled** (`stage/PixiStageCanvas.tsx`): canvas goes permanently
  blank on GPU reset while the ticker keeps painting.
- **Split-stack `itemInstanceId` can collide** (`inventory.ts:295`): `#split-${emptyIndex}`
  reuses indices after removals → duplicate ids, wrong-slot ops.
- **`buffStat` cost crashes on `undefined` payload** (`enemies/cost.ts:111`): uses
  `=== null` not `?? 0` like sibling cases.
- **`listRecent` ordering unstable on `created_at` ties** (`run-index.ts:75`) — bots use a
  fixed timestamp; add a `rowid` tiebreak.
- **`verify-ci.mjs`** crashes with uncaught ENOENT if `tests/` or `app/` is absent, and
  swallows `spawnSync.error` (prints FAIL with no cause).
- **Test artifacts pollute a tracked dir:** simulate/bot tests write into `runs/` (a tracked
  directory), so running the suite always leaves untracked artifacts behind. Add a gitignore
  rule for test-generated run dirs or redirect test output to a temp dir.

## False positives screened out (do not chase)

- **"Inverted quest-reward logic" (`machine.ts:90`)** — flagged Critical by an auditor.
  Not a bug: `statUpsAllowed`/`ruleBreaksAllowed` are config constants pinned to `false`
  (GAME_DESIGN §8); the branches are a dead fail-safe tripwire, harmless under real config.
- **"Quest-log `EntityId` vs `string` type violation" (Critical)** — typecheck passes;
  `EntityId` is a branded string, assignable to `string`. No error.
- **"Status duration off-by-one" (Critical)** — the code (`status.ts:291`,
  `duration-1 ≤ 0 → expire`) is correct.
- Note: `poison` intentionally floors HP at 1 (non-lethal) while `burn` can kill — a
  deliberate design asymmetry, not a bug, though burn's missing clamp makes lethality
  order-dependent when both + `regen` coexist.

## Recommended priority order

1. **C1** — one-line fix, prevents a silent total-generation outage.
2. **H1 + H2** — barter is broken whenever coin denomination > 1 (always); the merchant
   feature is effectively non-functional.
3. **H3** — one PRAGMA line; restores DB integrity.
4. **H4 / H5** — validation + replay completeness; both mask other failures.
